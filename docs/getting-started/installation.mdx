## Homebrew (recommended)

Homebrew is the easiest way to install podkit. It installs podkit and its only runtime dependency (FFmpeg) in a single command.

<Tabs>
  <TabItem label="One-liner">
    ```bash
    brew install jvgomg/podkit/podkit
    ```
  </TabItem>
  <TabItem label="Two-step">
    ```bash
    brew tap jvgomg/podkit
    brew install podkit
    ```
  </TabItem>
</Tabs>

Verify the installation:

```bash
podkit --version
```

That's it — you're ready to go. Continue to [Quick Start](/getting-started/quick-start) to set up your first sync.

### What Homebrew installs

- **podkit** — the CLI binary with all native dependencies statically linked
- **FFmpeg** — installed automatically as a dependency (required for transcoding)

No other runtime dependencies are needed.

## Manual Download

If you don't use Homebrew, you can download a prebuilt binary from [GitHub Releases](https://github.com/jvgomg/podkit/releases).

1. Download the tarball for your platform (e.g., `podkit-darwin-arm64.tar.gz`)
2. Extract it:

   ```bash
   tar xzf podkit-*.tar.gz
   ```

3. Move the binary somewhere on your `PATH`:

   ```bash
   sudo mv podkit /usr/local/bin/podkit
   ```

4. Verify it works:

   ```bash
   podkit --version
   ```

### Install FFmpeg manually

Manual installs do not include FFmpeg. You need to install it separately for transcoding to work.

<Tabs>
  <TabItem label="macOS">
    ```bash
    brew install ffmpeg
    ```
  </TabItem>
  <TabItem label="Ubuntu / Debian">
    ```bash
    sudo apt update && sudo apt install -y ffmpeg
    ```
  </TabItem>
  <TabItem label="Fedora">
    ```bash
    sudo dnf install -y \
      https://download1.rpmfusion.org/free/fedora/rpmfusion-free-release-$(rpm -E %fedora).noarch.rpm
    sudo dnf install -y ffmpeg
    ```
  </TabItem>
  <TabItem label="Arch">
    ```bash
    sudo pacman -S ffmpeg
    ```
  </TabItem>
</Tabs>

Check that FFmpeg has AAC encoder support:

```bash
ffmpeg -encoders 2>/dev/null | grep aac
```

You should see at least one AAC encoder listed. On macOS, you'll typically see both `aac` (native) and `aac_at` (AudioToolbox).

## Supported Devices

podkit works with classic iPods that use USB Mass Storage mode. iOS devices (iPod Touch, iPhone, iPad) are **not** supported. See [Supported Devices](/devices/supported-devices) for the full compatibility matrix.

:::note[Windows support]
Windows is not currently supported. This feature is on the [roadmap](/roadmap/) — vote and comment on the [discussion](https://github.com/jvgomg/podkit/discussions/8) to help us prioritise.
:::

## Docker

podkit is available as a Docker image for `linux/amd64` and `linux/arm64`, based on Alpine Linux with FFmpeg included.

```bash
docker run --rm -v ./config:/config ghcr.io/jvgomg/podkit:latest init
```

See the [Docker guide](/getting-started/docker) for full setup instructions with Docker Compose examples.
